# 1.小测的收获

## 零、 本地终端调试“名场面”避坑

### 1. 终端卡死/乱码假象

- **误区**：在终端输入代码或按回车后，屏幕被各种系统进程信息刷屏（不小心运行了 `top` 命令）。
    
- **解决**：用 `Ctrl + C` 强制中断进程。
    
- **误区**：程序没有任何反应，回车只能换行。
    
- **真相**：程序死循环了（例如 `while` 循环忘记写步长更新 `j++`），或者正在无休止地等待输入。按 `Ctrl + C` 强制中断进程。
    

### 2. 优雅地进行多行输入

- **手动结束输入**：当使用 `while(cin >> x)` 读取不知数量的多行数据时，程序不会因为你按了回车就停止。必须在新的一行按下 **`Ctrl + D`**（Mac）或 **`Ctrl + Z`**（Windows）发送 EOF（文件结束符）信号。
- **输入完了输出在哪里**：vscode中，输入完了之后输出会自动跳出来，并且往往没换行。

## 一、 输入流（I/O）与缓冲区

这是新手期最容易翻车的模块，核心在于理解 `cin` 和缓冲区的工作机制。

### 1. `cin >>` 类型的贪婪与残留

- **工作原理**：**自动跳过前导空格/换行**，读取连续字符，**遇到空格、Tab、换行符立即停止**
    
- **致命缺陷**：它会把导致它停止的那个空白字符（如换行符 `\n`）**留在输入缓冲区里**。
    
- **误区重现**：试图用 `cin >> arr` 去读取 `"push 10"`。
    
    - **错因**：`cin` 遇到 `push` 后面的空格就停了，`arr` 里只有 `"push\0"`，数字 `10` 根本没进数组！
        
    - **正确做法**：顺水推舟，分两次读：`cin >> cmd >> val;`。
        

### 2. `getline()` 类型的被动吞噬

- **工作原理**：**不会自动跳过前导空格/换行**，读取包括空格在内的一整行，**直到遇到换行符 `\n` 停止，并吃掉换行符**

- **getline() 基本语法：string类用 getline(cin,str) 而字符数组用 cin.getline(arr,100)

**👉 情况 A：如果用 `string`（强烈推荐）** 使用全局函数 `getline()`，只需要传两个参数。

```c++
string s;
getline(cin, s); // 直接搞定，完美读入一整行
```

**👉 情况 B：如果你用的是字符数组 `char[]`** 不能用上面的写法！必须使用 `cin` 自带的成员函数 `cin.getline()`，并且**必须强制要求你传入最大读取长度**（为了防越界）。

```c++
char arr[100];
// 参数：(数组名, 最大读取字符数)
// 它最多读取 99 个字符，最后 1 个位置自动留给 '\0'
cin.getline(arr, 100); 
```

- **getline不会跳过缓冲区！**：

    ```c++
    int n; string s;
    cin >> n;        // 读走数字，留下 '\n'
    getline(cin, s); // 迎面撞上 '\n'，直接返回空字符串！
    ```
    
- **解决规范**：若要在 `cin >>` 之后 `getline`，必须加一行清空缓冲区的代码：
    
    ```c++
    cin >> n;
    cin.ignore(); // 吞掉残留的换行符
    ```
    
- 🌟 总结复习（千万别忘的“拔剑动作”）

不管是哪种 `getline`，只要你在它**之前**使用了普通的 `cin >>`（比如读了组数 $n$），都请千万记得在中间加一句 **`cin.ignore();`** 去吃掉那个要命的换行符，否则你的 `getline` 就会直接读到一个空串！

### 3. `while (cin >> x)` 应该用在哪里？

- **误区**：在已知循环次数的 `for` 循环内部，又套了一个 `while(cin >> x)`。这会导致内层 `while` 一次性吃光所有测试样例。
    
- **正确情境**：**只在**题目没有给定总组数，要求“读到文件结束 (EOF)”时，才作为最外层循环使用。

### 4.string类的输入or字符数组的输入：都是cin >> 类型

- 都是跳过缓冲区，读取一连串的字符，遇到空格 or 换行符停止读取，遵循cin >> 的语法

- string类非常好用！例如下面这一段代码，如果用字符数组则要一个一个字符比对
```c++
string op;
cin >> op;//string的输入是直接到空格or换行结束，也就是截取一连串的字符
if (op == "push"){
	int x;
	cin >> x;
	sta.push(x);
}
```
### 5.总而言之
- 不要读入空格：用 cin >> 类型语法，推荐使用string类而不是字符数组
- 需要读入空格：用 getline() 类型语法，

## 二、 数组越界与死循环：逻辑杀手

### 1. 死循环：被遗忘的 `j++`

- **错例**：
    
    ```c++
    int j = 0;
    while(arr[j] != '\0') {
        // ... 逻辑判断 ...
        // 忘记写 j++; 导致永远在检查 arr[0]，程序卡死
    }
    ```
    
- **最佳实践**：遍历数组首选 `for` 循环 `for (int j = 0; arr[j] != '\0'; j++)`，把步长更新锁死在循环头上，防呆设计。

- ==在vscode里面，不会提示死循环！所以你的输入始终没有回报！==

### 2. 数组越界（Segmentation Fault）

- **错例 （找最小值逻辑错位）**：
    
    ```c++
    int tem = top_index;
    while(tem != -1) {
        tem--; // 如果此时 tem 是 0，自减后变成 -1
        if(data[tem] < min) // 越界访问 data[-1]，程序崩溃！
    }
    ```

## 三、 C风格字符串 vs C++ `std::string`

### 1. 字符数组 `char[]` (C 风格)

- **本质**：连续的内存块，必须以**隐藏字符 `'\0'`** 作为结束标志。
    
- **缺点**：长度不可变，容易越界；不能直接用 `==` 判断相等（必须用 `<cstring>` 库里的 `strcmp(a, b) == 0`）；不能用 `+` 拼接。
    

### 2. `std::string` (现代 C++ 利器)

- **包含库**：`#include <string>`
    
- **绝对优势**：自动扩容管理内存；直接用 `==` 判断相等；直接用 `+` 拼接；自带获取长度函数。
    
- **范围 for 循环（极其优雅的遍历）**：
    
    ```c++
    for(char ch : s) {
        // 自动从头到尾取出每一个字符，绝不会越界
    }
    ```
表示从s里面从头开始，遍历拿出每一个ch。

其中，ch : s可以理解为“ch in s”
